# 서비스 기획: 세계관/자캐/창작 공유 플랫폼

## 서비스 개요

이 프로젝트는 여러 사용자 또는 한 명의 사용자가 하나의 세계관을 제작하고, 그 세계관에 속한 자작 캐릭터를 만들며, 캐릭터의 소속과 관계도, 세계관 기반 창작물을 함께 관리하고 공유하는 서비스다.

서비스는 창작자가 자신의 세계관, 자캐, 소설, 설정놀이를 체계적으로 정리할 수 있는 개인 창작 도구이면서, 친구나 공동 창작자와 같은 세계관을 함께 발전시키고 공개적으로 공유할 수 있는 커뮤니티형 창작 플랫폼을 목표로 한다.

## 주요 고객

- 친구들과 자캐 설정을 함께 만들고 공유하고 싶은 사용자
- 자신의 독창적인 세계관과 캐릭터를 공개 및 공유하고 싶은 창작자
- 세계관 설정, 캐릭터 관계, 소설/설정놀이 기록을 흩어지지 않게 관리하고 싶은 사용자

## 핵심 가치

- 세계관, 자캐, 소속, 관계도, 창작물을 하나의 흐름으로 연결해 관리한다.
- 혼자 쓰는 설정 노트와 함께 만드는 공동 창작 공간을 모두 지원한다.
- 공개 공유, 특정 사용자 공유, 비공개 보관을 구분해 창작자의 공개 범위를 세밀하게 제어한다.
- 캐릭터 관계도와 창작물 기록을 통해 세계관의 변화와 캐릭터 서사를 추적할 수 있게 한다.

## 기술 스택

- Platform: Next.js, 현재 폴더 기준
- Database: PostgreSQL
- ORM: Prisma 7.8.0
- Auth: NextAuth session 방식

구현 전에는 반드시 `node_modules/next/dist/docs/`에서 현재 설치된 Next.js 버전에 맞는 문서를 확인한다. 이 프로젝트의 Next.js는 기존 지식과 다른 API, 구조, 규칙을 가질 수 있다.

## 핵심 기능

### 로그인 / 회원가입

- NextAuth session 방식을 사용한다.
- 이메일/비밀번호 또는 OAuth 제공자 지원 여부는 구현 단계에서 결정한다.
- 로그인 사용자는 자신의 세계관, 자캐, 창작물, 관계도를 생성하고 관리할 수 있다.
- 모든 쓰기 요청은 서버 측에서 세션을 검증해야 한다.

### 세계관 관리 CRUD

- 세계관 생성, 조회, 수정, 삭제를 지원한다.
- 세계관은 제목, 소개, 장르, 공개 범위, 소유자, 공동 작업자 목록, 캐릭터 소속 목록을 가진다.
- 공개 범위는 최소한 `private`, `shared`, `public` 세 가지를 고려한다.
- 세계관 안에서 사용할 소속 단위는 국가, 조직, 진영, 가문, 학원, 길드, 종족, 세력 등으로 확장될 수 있다.
- 세계관 삭제 시 관련 캐릭터, 캐릭터 소속, 관계도, 창작물 처리 정책을 명확히 정한다. MVP에서는 소유자만 삭제 가능하게 한다.

### 자캐 관리 CRUD

- 자캐는 특정 세계관에 소속된다.
- 캐릭터 생성, 조회, 수정, 삭제를 지원한다.
- 기본 필드는 이름, 별칭, 프로필 이미지, 한 줄 소개, 상세 설정, 성격, 배경, 태그, 공개 범위를 고려한다.
- 캐릭터는 0개 이상의 세계관 내 소속에 연결될 수 있고, 소속별 직책, 계급, 상태, 가입/탈퇴 시점, 비고를 가질 수 있다.
- 대표 소속을 지정할 수 있어 캐릭터 목록과 관계도에서 캐릭터의 현재 중심 소속을 빠르게 보여준다.
- 캐릭터는 세계관 권한을 따르되, 필요하면 캐릭터별 공개 범위를 추가로 둘 수 있다.

### 캐릭터 소속 관리 CRUD

- 세계관 안에서 캐릭터가 속할 수 있는 소속을 생성, 조회, 수정, 삭제한다.
- 소속은 이름, 유형, 설명, 상징 이미지, 대표 색상, 공개 범위, 정렬 순서를 가질 수 있다.
- 캐릭터와 소속의 연결을 생성, 조회, 수정, 삭제한다.
- 한 캐릭터는 여러 소속에 속할 수 있으며, 소속 연결에는 직책, 계급, 현재/과거 상태, 시작일, 종료일, 메모를 기록할 수 있다.
- 소속 삭제 시 연결된 캐릭터 소속 이력을 어떻게 처리할지 명확히 정한다. MVP에서는 소속 삭제 시 캐릭터-소속 연결을 함께 삭제하거나, 삭제 전 연결 해제를 요구하는 방식을 선택한다.
- 소속과 캐릭터는 반드시 같은 세계관에 속해야 하며, 서버 측에서 이 제약을 검증한다.

### 세계관 및 자캐 기반 소설/설정놀이 관리 CRUD

- 세계관에 속한 소설, 설정놀이, 단편 기록, 에피소드를 생성하고 관리한다.
- 창작물은 관련 세계관, 등장 캐릭터, 관련 소속을 연결할 수 있어야 한다.
- 창작물 타입은 `novel`, `roleplay`, `setting_note` 같은 구분을 고려한다.
- 창작물은 특정 소속 중심의 에피소드, 세력 간 갈등, 조직 내부 설정놀이처럼 소속을 기준으로 분류하고 탐색할 수 있어야 한다.
- 창작물은 특정 등장 캐릭터 중심을 기준으로 분류하고 탐색할 수 있어야 한다.
- 긴 글 관리를 위해 본문 단일 저장 방식과 챕터/에피소드 분리 방식을 비교한다. MVP에서는 `Work`와 `WorkChapter` 구조를 권장한다.

### 특정 사용자와 세계관 공유

- 세계관 소유자는 특정 사용자를 초대하거나 공유할 수 있다.
- 권한은 `owner`, `admin`, `editor`, `viewer`를 기본 후보로 둔다.
- MVP에서는 `owner`, `editor`, `viewer`만으로 시작해도 된다.
- 공유받은 사용자는 권한에 따라 세계관, 자캐, 캐릭터 소속, 창작물, 관계도를 읽거나 수정할 수 있다.
- 소속 생성/수정/삭제와 캐릭터 소속 연결 변경은 세계관 편집 권한이 있는 사용자만 수행한다.

### 자캐 관계도 관리 CRUD

- 같은 세계관에 속한 캐릭터 간 관계를 생성, 조회, 수정, 삭제한다.
- 관계는 출발 캐릭터, 대상 캐릭터, 관계명, 설명, 방향성, 상태를 가진다.
- 예: 친구, 가족, 연인, 적대, 스승/제자, 과거 인연.
- 관계는 캐릭터의 현재 소속이나 특정 소속 맥락을 함께 참조할 수 있다.
- 관계도 화면에서는 노드가 캐릭터, 엣지가 관계가 되는 그래프 표현을 목표로 하며, 소속별 그룹핑, 필터링, 색상 표시를 지원하는 방향으로 설계한다.

## 권한 모델

- `owner`: 세계관 생성자. 모든 관리 권한과 삭제 권한을 가진다.
- `admin`: 공동 관리자. 멤버 관리와 주요 편집 권한을 가진다.
- `editor`: 세계관 하위 리소스인 캐릭터, 캐릭터 소속, 관계, 창작물을 생성/수정할 수 있다.
- `viewer`: 공유받은 세계관과 하위 리소스를 읽을 수 있다.

권한 검사는 클라이언트 UI에서만 처리하지 않고, 서버 액션/API/라우트 핸들러 등 모든 서버 측 쓰기 경로에서 반복 검증한다.

## 추천 도메인 모델

아래 모델은 구현 시작 전 기준이 되는 초안이다. Prisma 스키마 작성 시 실제 NextAuth adapter 요구사항과 Prisma 7.8.0 문서를 함께 확인한다.

- `User`: 서비스 사용자
- `World`: 세계관
- `WorldMember`: 세계관 공유 및 권한
- `Character`: 세계관에 속한 자캐
- `Affiliation`: 세계관 안에서 캐릭터가 속할 수 있는 소속, 조직, 진영, 가문, 국가 등
- `CharacterAffiliation`: 캐릭터와 소속의 연결 및 소속 이력
- `CharacterRelation`: 자캐 간 관계
- `Work`: 소설, 설정놀이, 설정 노트 등 창작물 묶음
- `WorkChapter`: 창작물의 챕터/에피소드/로그 단위
- `WorkCharacter`: 창작물과 등장 캐릭터의 연결 테이블
- `WorkAffiliation`: 창작물과 관련 소속의 연결 테이블
- `Tag`: 세계관, 자캐, 창작물 검색과 분류에 사용할 태그

## 주요 화면

- 홈/탐색: 공개 세계관과 공개 창작물을 탐색한다.
- 내 작업실: 내가 소유하거나 공유받은 세계관 목록을 본다.
- 세계관 상세: 소개, 캐릭터 목록, 소속 목록, 관계도, 창작물 목록, 멤버를 관리한다.
- 소속 관리: 세계관 내 조직, 진영, 국가, 가문 등 소속을 생성하고 캐릭터 연결을 관리한다.
- 캐릭터 상세: 프로필, 상세 설정, 소속 이력, 등장 창작물, 관계 목록을 본다.
- 관계도 편집: 캐릭터 노드와 관계 엣지를 생성하고 수정하며, 소속별 표시와 필터를 조정한다.
- 창작물 편집: 소설/설정놀이 본문과 연결 캐릭터, 관련 소속을 관리한다.
- 공유 관리: 사용자 초대, 권한 변경, 공유 해제를 처리한다.

## MVP 범위

- 회원가입/로그인/로그아웃
- 내 세계관 CRUD
- 세계관별 캐릭터 소속 CRUD
- 세계관별 캐릭터 CRUD
- 캐릭터 관계 CRUD
- 세계관별 창작물 CRUD
- 특정 사용자 공유 및 권한 기반 접근 제어
- 공개/비공개 세계관 조회

## 이후 확장 후보

- 캐릭터 관계도 그래프 뷰 고도화
- 소속별 타임라인, 계급도, 조직도 뷰
- 공동 편집 이력 및 변경 로그
- 댓글, 좋아요, 북마크
- 세계관 템플릿 및 캐릭터 프로필 템플릿
- 이미지/자료 첨부
- 작품 연재, 회차 공개 예약
- 검색, 태그, 장르별 탐색
- 신고/차단/콘텐츠 공개 정책

## 구현 원칙

- `docs/ui`의 HTML이 화면 기준으로 지정되면 해당 파일의 DOM 구조, 간격, 크기, 색상, 반응형 동작과 상호작용을 구현 기준으로 삼는다.
- `docs/ui` HTML 파일을 iframe, 정적 파일 링크, 리다이렉트 등으로 직접 연결하지 않는다. Next.js 페이지와 재사용 가능한 React 컴포넌트, Tailwind CSS로 동일하게 재구현한다.
- 세계관을 최상위 소유/공유 단위로 보고, 하위 리소스 권한은 기본적으로 세계관 권한을 상속한다.
- 캐릭터, 소속, 관계, 창작물은 모두 같은 세계관 경계 안에서 연결되어야 하며, 서로 다른 세계관의 리소스가 연결되지 않도록 서버 측에서 검증한다.
- 삭제나 권한 변경처럼 영향 범위가 큰 작업은 서버 측에서 소유자 또는 관리자 권한을 엄격히 확인한다.
- Prisma 관계는 명시적으로 설계해 캐릭터, 소속, 관계, 창작물 간 연결을 쉽게 조회할 수 있게 한다.
- 사용자 입력 본문은 XSS와 권한 누출을 고려해 저장/렌더링 정책을 정한다.
- 공개 데이터와 비공개 데이터가 섞이는 목록 조회에서는 쿼리 단계에서 접근 가능한 리소스만 반환한다.
