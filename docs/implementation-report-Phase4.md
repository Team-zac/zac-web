# Phase 4 구현 보고서

## 범위

Phase 4에서는 세계관 내부 캐릭터와 소속 CRUD, 캐릭터 태그, 캐릭터-소속 연결 이력을 실제 Prisma 데이터와 연결했다. 화면은 `docs/ui/12`, `13`, `14`, `15`, `16`, `17`, `18`, `19`, `26` 목업의 섹션 순서, 카드, 선택 UI, CTA 디자인을 기준으로 구성했다.

## 구현 내용

### 캐릭터 CRUD

- 캐릭터 생성 및 수정 폼 구현
  - 세계관 가로 선택 카드
  - 이름, 별칭, 프로필 이미지 URL
  - 한 줄 소개
  - 공개 범위
  - 태그
  - 성격, 배경
  - Markdown 상세 소개와 문법 삽입 버튼
- 생성 및 수정 Server Action에서 세션과 세계관 편집 권한 재검증
- 캐릭터 수정 시 소속 세계관 변경 차단
- 목록에서 선택 삭제 모드, 취소, 확인 모달 구현
- 삭제는 논리 삭제로 처리

### 캐릭터 태그

- 입력 태그의 `#` 제거
- 공백과 쉼표 기준 분리
- 소문자 정규화와 중복 제거
- 세계관 범위 태그 upsert
- 캐릭터 수정 시 기존 연결을 새 입력값으로 동기화

### 소속 CRUD

- 소속 생성 및 수정 폼 구현
  - 세계관 선택
  - 이름
  - 소속 유형
  - 상징 이미지 URL
  - 대표 색상
  - 공개 범위
  - Markdown 설명과 문법 삽입 버튼
  - 상위 소속 가로 선택 카드
- 대표 색상 `#RRGGBB` 형식 검증
- 상위 소속의 동일 세계관 제약 검증
- 자기 자신 선택과 소속 계층 순환 구조 방지
- 목록 선택 삭제와 확인 모달 구현
- 소속 삭제 시 캐릭터 연결 제거, 하위 소속의 상위 연결 해제, 소속 논리 삭제

### 캐릭터-소속 연결

- 같은 세계관의 소속만 연결 가능
- 다중 소속 선택 카드 구현
- 선택된 각 소속별 연결 정보 편집
  - `title`
  - `rank`
  - `status`
  - `startedLabel`
  - `endedLabel`
  - `note`
  - `isPrimary`
- 대표 소속은 하나만 지정 가능
- 캐릭터 수정 화면에서 기존 연결값과 대표 소속 상태 복원

### 캐릭터 상세

- 소개, 태그, 별칭
- 등장 세계관 카드
- 대표 소속과 직책, 계급, 상태
- Markdown 상세 소개
- 성격과 배경
- 등장 창작물 가로 카드와 빈 상태
- 해당 캐릭터 중심 관계 영역과 빈 상태
- 편집 권한 사용자에게 수정, 소속 관리, 창작물 만들기 CTA 노출

### 소속 상세

- 상징 이미지 또는 대표 색상을 사용한 기본 상징
- 대표 색상 미리보기
- 소속 유형과 공개 범위
- Markdown 설명
- 상위 소속
- 현재, 과거 상태와 대표 여부를 포함한 소속 멤버 카드
- 편집 권한 사용자에게 수정 CTA 노출

### 공개 캐릭터 탐색

- 캐릭터와 세계관이 모두 PUBLIC인 데이터만 조회
- 이름, 별칭, 소개, 세계관, 소속, 태그 통합 검색
- 인기순과 최신순 정렬
- 페이지당 15개 페이지네이션
- 검색 초기화와 빈 결과 화면

## UI 검증

요청에 따라 `npm run build`, `npm run lint`는 실행하지 않고 `localhost:3000`을 Chrome으로 직접 열어 검증했다.

- `/characters/new`
  - 편집 가능한 세계관이 없을 때 빈 상태 확인
  - 세계관 생성 후 세계관 선택 카드와 전체 입력 필드 렌더링 확인
- `/affiliations/new?worldId=...`
  - PUBLIC 소속 생성
  - 유형, 대표 색상, Markdown 설명 저장
  - 생성 후 소속 상세 이동
- `/characters/new?worldId=...`
  - PUBLIC 캐릭터 생성
  - 태그와 Markdown 소개 저장
  - 소속 선택 및 직책, 계급, 상태, 시작 시점, 메모, 대표 소속 저장
- `/characters/[characterId]`
  - 등장 세계관, 대표 소속, 연결 정보, Markdown 소개 렌더링
  - 창작물과 관계 빈 상태 확인
- `/characters/[characterId]/edit`
  - 기존 캐릭터 값, 선택 소속, 직책, 대표 소속 상태 복원
  - 수정 저장 후 상세 화면 이동
- `/affiliations/[affiliationId]`
  - 상징, 대표 색상, Markdown 설명, 상위 소속, 소속 멤버 카드 확인
- `/affiliations/[affiliationId]/edit`
  - 소속명 수정 후 상세 화면 반영
- `/characters`
  - 선택 삭제 모드와 확인 모달 확인
  - 캐릭터 논리 삭제 성공
- `/affiliations`
  - 선택 삭제와 연결 정리 성공
- `/characters/explore`
  - 생성한 PUBLIC 캐릭터, 세계관, 대표 소속, 태그 노출
- 390px 모바일
  - 캐릭터 상세의 카드 단일 열 전환
  - 하단 내비게이션 유지
  - 문서 가로 오버플로 없음
