# Phase 5 구현 보고서

## 범위

Phase 5에서는 캐릭터 관계 CRUD와 그래프 기반 조회/편집 화면을 구현했다. 화면은 `docs/ui/20-relations-graph.html`의 페이지 히어로, 관계 그래프 패널, 드래그 캔버스, 간선 선택 수정 모달, 관계 추가 모달, 관계 삭제 CTA를 기준으로 구성했다.

## 구현 내용

### 관계 CRUD

- 관계 생성 Server Action 구현
  - 출발 캐릭터
  - 대상 캐릭터
  - 관계명
  - 설명
  - 방향성
  - 상태
  - 공개 범위
  - 관계 맥락 소속
- 관계 수정 Server Action 구현
- 관계 삭제 Server Action 구현
- 삭제는 논리 삭제로 처리
- 모든 쓰기 액션에서 로그인 사용자와 세계관 편집 권한을 재검증

### 서버 검증

- 출발 캐릭터와 대상 캐릭터가 서로 다른지 검증
- 두 캐릭터가 같은 세계관에 속하는지 검증
- 관계 맥락 소속이 같은 세계관에 속하는지 검증
- 관계 수정 시 관계의 세계관 변경 차단
- 관계 삭제 시 관계가 속한 세계관의 편집 권한 확인

### 그래프 데이터

- 세계관 기준 그래프 데이터 조회 함수 구현
- 노드 데이터
  - 캐릭터 ID
  - 캐릭터 이름
  - 대표 소속 이름
  - 대표 소속 색상
  - 그래프 좌표
- 엣지 데이터
  - 관계 ID
  - 출발/대상 캐릭터
  - 관계명
  - 상태
  - 방향성
  - 공개 범위
  - 관계 맥락 소속
  - 설명
  - 간선 좌표, 길이, 회전각
- 데이터 조회는 캐릭터 100개, 관계 300개까지 제한해 Phase 5 성능 기준을 반영
- 캐릭터 상세 재사용을 위해 특정 캐릭터를 중앙 노드로 배치하는 옵션 추가

### 관계도 UI

- `/relations?worldId=...` 실제 관계도 화면 구현
- `#new` 해시로 진입 시 관계 추가 모달 자동 오픈
- 캔버스 드래그 이동 구현
- 캐릭터 노드 표시
- 관계 간선 표시
- 간선 클릭 시 선택 관계 수정 모달 오픈
- 관계 추가 모달 구현
- 관계 삭제 CTA 구현
- 편집 권한이 없으면 읽기 전용 칩 표시
- 관계 추가, 저장, 삭제 버튼은 편집 권한이 있는 사용자에게만 노출
- 공통 컴포넌트 사용
  - `Panel`
  - `Button`
  - `Chip`
  - `BackButton`

### 재사용 연결

- 세계관 상세의 관계 그래프를 실제 관계 데이터 기반 `RelationGraph`로 교체
- 캐릭터 상세의 관계 영역을 같은 `RelationGraph`로 교체
- 캐릭터 상세에서는 현재 캐릭터가 중앙에 오도록 그래프 데이터 옵션 적용

## UI 검증

요청에 따라 `npm run build`, `npm run lint`는 실행하지 않고 `localhost:3000`을 Chrome으로 직접 열어 검증했다.

- `/relations?worldId=884ee947-f44a-4d13-bb03-6313aef9f5c8#new`
  - HTTP 렌더링 정상
  - 오류 오버레이 없음
  - 읽기 전용 그래프 렌더링 확인
- 테스트 세계관 생성
  - Phase 5 검증용 세계관 생성
  - 캐릭터 2명 생성
- `/relations?worldId=...#new`
  - `#new` 진입 시 관계 추가 모달 자동 오픈
  - 관계 생성 성공
  - 생성 후 그래프 간선 1개 표시
- 간선 클릭 수정 모달
  - 관계명 `과거 인연` 표시 확인
  - 관계명 `감시 대상`으로 수정
  - 상태 `RUMORED`로 수정
  - 수정 후 모달 재오픈 시 수정값 반영 확인
- 관계 삭제
  - 관계 삭제 CTA 실행
  - 삭제 후 간선 개수 0개 확인
- 390px 모바일 관계도
  - 페이지 렌더링 정상
  - 문서 가로 오버플로 없음
- 검증용 세계관과 하위 캐릭터, 관계 데이터 정리 완료
