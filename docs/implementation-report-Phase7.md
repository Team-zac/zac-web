# Phase 7 구현 보고서

## 범위

Phase 7에서는 내 작업실과 localStorage 기반 임시 저장 MVP를 구현했다. 화면은 `docs/ui/05-workspace.html`의 페이지 히어로, 요약 지표, 임시 저장 가로 카드, 세계관/캐릭터/창작물 가로 카드 섹션, 로그아웃 링크 구조를 기준으로 구성했다.

## 구현 내용

### 내 작업실

- `/workspace`를 샘플 데이터에서 실제 사용자 데이터 기반 화면으로 교체
- 로그인하지 않은 사용자는 `/login`으로 이동
- 로그인 사용자 표시
- 요약 지표 구현
  - 접근 가능한 세계관 수
  - 접근 가능한 캐릭터 수
  - 접근 가능한 창작물 수
- 세계관 가로 카드 섹션 구현
- 캐릭터 가로 카드 섹션 구현
- 창작물 가로 카드 섹션 구현
- 각 섹션에 보기/제작 CTA 연결
- 실제 접근 가능한 데이터만 조회
  - 본인 소유 세계관
  - 공유받은 세계관
  - 접근 가능한 세계관의 캐릭터
  - 접근 가능한 세계관의 창작물

### 임시 저장

- localStorage 기반 초안 인덱스 구현
- `/workspace` 임시 저장 섹션에서 저장된 초안 목록 표시
- 초안 카드는 저장 시간, 리소스 유형, 제목, 설명을 표시
- 초안 클릭 시 해당 작성 화면으로 이동
- `draftKey` query를 통해 저장된 입력값 복구

### 자동 저장 적용 화면

- 세계관 작성 화면
- 소속 작성 화면
- 캐릭터 작성 화면
- 창작물 작성 화면
- 챕터 편집 화면

### 저장/복구 정책

- 입력 변경 후 짧은 지연을 두고 localStorage에 자동 저장
- 저장 대상은 현재 작성 URL 기준으로 분리
- 초안 목록은 `zac:drafts:index`에 최신순으로 유지
- 작성 화면 submit 시 해당 초안을 제거
- 챕터 편집기는 일반 form 구조가 아니므로 상태 기반 별도 자동 저장을 적용

### 로그아웃

- `/workspace` 하단 로그아웃 CTA를 NextAuth `signOut` 호출로 연결
- 로그아웃 후 시작 화면으로 이동

### 공통 컴포넌트 적용

- `PageShell`
- `ButtonLink`
- `Chip`
- 신규 `WorkspaceDrafts`
- 신규 `DraftAutoSave`
- 신규 `LogoutButton`

## UI 검증

요청에 따라 `npm run build`, `npm run lint`는 실행하지 않고 `localhost:3000`을 Chrome으로 직접 열어 검증했다.

- `/workspace`
  - 실제 사용자 이름 표시 확인
  - 세계관/캐릭터/창작물 실제 수치 표시 확인
  - 실제 세계관/창작물 카드 표시 확인
  - 캐릭터 빈 상태 표시 확인
  - 오류 오버레이 없음
- `/worlds/new`
  - 세계관 제목, slug, 소개 입력
  - localStorage 초안 저장 확인
- `/workspace`
  - 임시 저장 섹션에 세계관 초안 표시 확인
- 임시 저장 카드 클릭
  - `/worlds/new?draftKey=...` 이동 확인
  - 제목, slug, 소개 복구 확인
- `/works/[workId]/chapters`
  - 챕터 본문 수정
  - localStorage 챕터 초안 저장 확인
- `/workspace`
  - 임시 저장 섹션에 챕터 초안 표시 확인
